# Chapter Four

There are multiple methods for test case design, it mainly consists of 2 categories, Blackbox testing and whitebox testing

Black box testing includes:
- Equivalence Partitioning


## Equivalence Partitioning
Is simple reducing the number of testcases for the minimum number of subsets needed to cover all test data


![image.png](/images/paste-1773045331333-610c0777a46a348b.png)

How to define equivalence classes:
- if its a range its usually 3 classes (Valid and 2 invalid)
- if member of a set or a boolean it is 2 (1 valid and 1 invalid)

## Boundary value analysis

It improves EP, we test 6 values instead of 3 to make sure the boundaries are correct
